# Bloc & Cubit trong Flutter

Trong **ShiftFlow**, chúng ta dùng `flutter_bloc` để tách biệt UI ra khỏi logic (Business Logic Component). Cụ thể, thay vì dùng `setState()` dài dòng và khó quản lý khi app lớn lên, chúng ta dùng **Cubit** (một dạng đơn giản hóa của BLoC).

## 1. Cubit là gì?

Cubit là class lưu trữ state (trạng thái). UI sẽ lắng nghe (listen) state này, mỗi khi state thay đổi, UI sẽ tự động vẽ lại.

```dart
// Ví dụ Cubit đơn giản:
class CounterCubit extends Cubit<int> {
  CounterCubit() : super(0); // Bắt đầu ở số 0

  void increment() => emit(state + 1); // Phát (emit) ra state mới
}
```

## 2. BaseCubit trong ShiftFlow

Thay vì viết lại code loading, success, error nhiều lần, **ShiftFlow** cung cấp `BaseCubit`:

```dart
class BaseCubit<S> extends Cubit<S> {
  // Cung cấp sẵn các helper:
  void setLoading() { /* ... */ }
  void setSuccess() { /* ... */ }
  void setError(String message) { /* ... */ }
}
```

Trạng thái (State) thường đi với `Freezed` (xem thêm file `3.freezed.md`).

## 3. Cách sử dụng Cubit với UI

### Bước 1: Cung cấp (Provide) Cubit
Để màn hình dùng được Cubit, ta phải bọc màn hình bằng `BlocProvider`:

```dart
BlocProvider(
  create: (context) => getIt<NotificationCubit>()..loadNotifications(),
  child: NotificationPage(),
)
```

### Bước 2: Lắng nghe và Vẽ UI (BlocBuilder)
Ở dưới widget con, dùng `BlocBuilder` để vẽ theo state:

```dart
BlocBuilder<NotificationCubit, NotificationState>(
  builder: (context, state) {
    if (state.status == BaseStatus.loading) {
      return CircularProgressIndicator();
    }
    
    // Khi lấy được danh sách
    return ListView(
      children: state.notifications.map(...).toList(),
    );
  },
)
```

### Bước 3: Gọi hàm trong Cubit từ UI
VD: khi người dùng ấn nút Refresh, ta bảo Cubit tải lại dữ liệu:

```dart
IconButton(
  icon: Icon(Icons.refresh),
  onPressed: () {
    context.read<NotificationCubit>().loadNotifications();
  },
)
```

---

## 🔥 Lưu ý cho người mới
- `BlocBuilder`: Chỉ dùng để **vẽ UI**. Hàm `builder` trả về Widget.
- `BlocListener`: Giống Builder, nhưng không vẽ UI, nó dùng để thực hiện **Action 1 lần** (như show SnackBar, chuyển trang khi đăng nhập thành công, show Alert,...).
- `BlocConsumer`: Kết hợp cả Builder và Listener.
- **Không bao giờ chèn code UI hay Navigation vào trong class Cubit!** Cubit chỉ quan tâm dữ liệu.
