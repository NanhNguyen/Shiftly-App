# Gọi mảng HTTP bằng Dio

Thay thế cho gói HTTP lởm khởm mặc định, ShiftFlow dùng **Dio**. 
Nó có ưu điểm: chặn request (Interceptor - để luôn gửi theo token), huỷ request, và convert Response tốt.

## 1. Bản chất Repository Pattern
Thay vì viết `dio.get` tùm lum ở màn hình, thì phải có quy chuẩn. UI không quan tâm là bạn lấy data bằng Dio hay File Local, nó chỉ mượn thằng Repo. 

- UI gọi Repo (`ScheduleRequestRepo`).
- `ScheduleRepoImpl` dùng Dio gửi data.

## 2. Khai báo API

Bên `app_config.dart` hoặc `di_config.dart` (mục module), nơi ta gắn config Dio ban đầu:

```dart
@module
abstract class AppModule {
  @lazySingleton
  Dio get dio {
    final dio = Dio(
      BaseOptions(
        baseUrl: 'https://shiftflow-app-backend.onrender.com/', // Đổi IP nhé
        connectTimeout: const Duration(seconds: 10),
      ),
    );

    // Bắt toàn bộ Request gửi đi, nhét Token vào đầu
    dio.interceptors.add(
      InterceptorsWrapper(
        onRequest: (options, handler) async {
         // final token = ... lấy token ra từ storage (shared_preferences...)
         // options.headers['Authorization'] = 'Bearer $token';
          return handler.next(options);
        },
      ),
    );
    return dio;
  }
}
```

## 3. Class Repository Implementation 
Vì có Injectable nên hàm constructor chỉ cần kêu gọi Dio!

```dart
@Injectable(as: AuthRepo)
class AuthRepoImpl implements AuthRepo {
  final Dio _dio; // Injectable tự kiếm thằng @lazySingleton Dio ở trên ném vào đây

  AuthRepoImpl(this._dio);

  @override
  Future<String> login(String email, String password) async {
    // 1. Gửi request dạng POST
    final response = await _dio.post(
      '/auth/login',
      data: {
        'email': email,
        'password': password,
      },
    );

    // 2. Map data thẳng từ JSON
    return response.data['access_token'];
  }
}
```

## Lưu ý về Catch Error!
Trong `BaseCubit` có hàm `setError`. Ta có thể bắt trực tiếp `DioException`:
```dart
try {
  // Lệnh call api dính lỗi 403 / 500
} on DioException catch (e) {
   final msg = e.response?.data['message'] ?? e.message;
   setError("Lỗi Server: $msg");
}
```
